### 📅 2026年3月21日 | 项目重构与性能优化总结

#### 1. 架构升级：Logic & State 分离
* **成就**：成功将 SSE（Server-Sent Events）的底层连接逻辑从 Pinia Store 中剥离，封装成了独立的 **Composable (`useSSE.ts`)**。
* **价值**：实现了逻辑复用。Store 现在只负责业务状态（数据给谁），而 Composable 负责脏活累活（怎么拿数据、怎么断开连接）。这符合 Vue 3 的最佳实践，极大提升了代码的可维护性。

#### 2. 核心突破：三大“金边”优化
为了让打字机效果达到工业级水准，我们合力完成了以下优化：
* **数据截断处理 (Leftover Buffer)**：通过建立缓冲区解决 TCP 分包导致的 `data: ` 前缀丢失问题，保证了流式数据在高并发/网络波动下的 **100% 完整性**。
* **分帧节流渲染 (rAF Performance)**：引入 `requestAnimationFrame` 机制，将 UI 更新频率锁定在 **60FPS**。避免了 AI 喷涌数据时 Vue 频繁重绘导致的 CPU 占用过高和页面卡顿。
* **资源安全解锁 (Resource Safety)**：引入 `AbortController` 和 `reader.releaseLock()`。确保在用户刷新、断网或切换任务时，能够主动掐断请求并释放浏览器内存锁，消灭了“僵尸连接”。


#### 3. 视觉与交互修复
* **Markdown 样式对齐**：通过引入 `github-markdown-css` 并配合 `:deep(.markdown-body)` 深度选择器，修复了任务描述渲染失效的问题，让界面回归了 GitHub 风格的极客感。
* **_Loading_ 状态接力**：优化了从“转圈等待”到“打字机喷涌”的无缝切换逻辑，消除了首字节到达前的感知延迟。
* **TS 类型安全**：通过局部变量快照（Snapshot）和非空断言，彻底消灭了 IDE 中关于 `task.value` 异步指向不明的红线报错。
