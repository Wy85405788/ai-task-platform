## 📝 开发者日志 (Development Log)

### 📅 2026年3月19日
**今日主题：项目工程化搬迁与数据持久化打通**

#### 🚀 核心进展
* **项目架构现代化 (Engineering)**
    * 完成了从单文件脚本向 **Vite + Vue 3 + TypeScript** 标准工程结构的全面迁移。
    * 引入了强类型定义 `interface AITask`，规范了前后端数据交互协议，大幅提升了代码的可维护性。
    * 集成了 **Tailwind CSS**，实现了响应式分栏布局，取代了臃肿的内联样式。

* **后端持久化升级 (Backend)**
    * 放弃了临时的内存存储，实现了基于 `tasks_db.json` 的**文件数据库系统**。
    * 开发了任务自动保存与加载逻辑，确保用户练习记录在服务器重启后依然完好无损。
    * 优化了 FastAPI 接口，支持历史记录查询 (`/task/history`) 和异步代码批改反馈保存。

* **前端 UI/UX 重构 (Frontend)**
    * **新增「历史挑战」侧边栏**：支持从后端实时拉取历史任务，并实现了任务点击回显（包括旧代码和 AI 反馈）。
    * **原生化去依赖**：将图标库从 `lucide-react` 替换为原生 **SVG**，提升了在各种开发环境下的编译兼容性。
    * **交互优化**：保留并调优了 Markdown 流式输出 (Streaming) 和输出自动置底 (Auto-scroll) 逻辑，确保了极佳的 AI 对话感。

#### 🛠️ 技术栈更新
* **Frontend**: Vue 3 (Setup), TypeScript, Tailwind CSS, Markdown-it
* **Backend**: FastAPI, Python, JSON Storage

#### 💡 经验总结
> “不容易，真不容易。”
> 今天的难点不在于写新功能，而是在**不破坏原有精细交互（如流式渲染）的前提下，对整套底层结构进行‘无损手术’**。成功的关键在于对数据结构的预先对齐和对 Vue 组件生命周期的精确控制。
