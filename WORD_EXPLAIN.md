### 📚 核心词汇科普：从“手动挡”到“自动挡”

当我们说要把数据从 JSON 迁移到数据库时，这三个词是你绕不开的“三剑客”：

#### 1. SQLite 数据库 🗄️
* **通俗解释**：它是一个**“文件即数据库”**的轻量级系统。普通的数据库（如 MySQL）需要安装软件、配置账号密码；而 SQLite 就像一个加强版的 `.txt` 或 `.json` 文件，但它支持标准的 SQL 查询语言。
* **🚩 面试考点**：它是**“无服务器 (Serverless)”**的。
    * **优点**：零配置，读速度极快，适合本地开发、手机 App 或中小型网站。
    * **缺点**：不适合极高并发的写入（因为它在写数据时会锁定整个文件）。

#### 2. ORM (Object-Relational Mapping) 映射器 🌉
* **通俗解释**：它是连接“代码世界”和“数据库世界”的**翻译官**。
    * **代码世界**：你习惯用 `task.id = 1` 这样的对象操作。
    * **数据库世界**：它只听得懂 `SELECT * FROM tasks...` 这种 SQL 语句。
* **🚩 面试考点**：**“解耦”与“开发效率”**。用了 ORM，你不需要手写复杂的 SQL 字符串，直接操作 Python 对象，ORM 会自动帮你生成 SQL 并在后台执行。

#### 3. SQLAlchemy 🛠️
* **通俗解释**：它是 Python 领域**最强大、最流行**的一个 ORM 框架。
* **🚩 面试考点**：它提供了**“数据库抽象层”**。这意味着你今天用 SQLAlchemy 写 SQLite，明天想换成大公司的 PostgreSQL，只需要改一行连接字符串，剩下的业务逻辑代码一行都不用动。
